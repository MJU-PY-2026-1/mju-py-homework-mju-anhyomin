반대로 권장 칼로리에 근접했을 경우에는 남은 칼로리 범위 안에서 섭취할 수 있는 다이어트 음식이나 식단을 추천해 줍니다.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정

### **🛠️ Troubleshooting & 기술 회고:**
  1. 문제 1: if문이 while True 반복문 밖으로 빠져 있었다.
   - 원인:처음 코드를 작성할 때 메뉴를 입력받는 부분까지만 while True 안에 넣고, 실제 메뉴를 처리하는 if-elif-else문은 반복문 밖에 작성하였다. 그 결과 break와 continue가 반복문 안에서 작동하지 못해 오류가 발생하였다.
   - 해결: if menu == "1":부터 else문까지 전체 메뉴 처리 코드를 while True 안쪽으로 들여쓰기 하였다. 이를 통해 사용자가 메뉴를 선택할 때마다 조건문이 반복 실행되도록 수정하였다.
2. 문제 2: menu 값을 숫자로 비교하여 조건문이 제대로 실행되지 않았다.
   - 원인:input() 함수로 입력받은 값은 문자열 자료형인데, 처음에는 menu == 1, menu == 2처럼 정수형 숫자와 비교하였다. 그래서 사용자가 1을 입력해도 조건이 참으로 판정되지 않았다.
   - 해결:menu == "1", menu == "2"처럼 입력값을 문자열로 비교하도록 수정하였다. 이를 통해 사용자가 입력한 메뉴 번호에 맞게 각 기능이 정상적으로 실행되었다.
3. 문제 3: BMI 계산 부분에서 들여쓰기 오류가 발생하였다.
   - 원인:bmi = weight / (height_meter ** 2) 아래의 if bmi >= 25: 코드가 같은 깊이로 정렬되지 않았다. 파이썬은 들여쓰기를 기준으로 코드 블록을 구분하기 때문에, 깊이가 맞지 않으면 문법 오류가 발생한다.
   - 해결: bmi 계산 코드와 BMI 판정 조건문들의 들여쓰기를 올바르게 맞추었다. 
4. 문제 4: 사용자 정보를 입력하기 전에 2, 3, 4번 메뉴를 선택하면 오류가 발생하였다.
   - 원인:사용자가 1번 메뉴를 먼저 선택하지 않으면 weight, height, goal_weight, information 값이 아직 만들어지지 않는다. 그런데 2번, 3번, 4번 기능에서는 이 값들을 사용하려 했기 때문에 오류가 발생할 수 있었다.
   - 해결:프로그램 시작 부분에 information = []을 먼저 작성하였다. 그리고 2번, 3번, 4번 메뉴 안에 len(information) == 0 조건을 넣어 저장된 정보가 없을 경우 “먼저 사용자 정보를 입력해주세요.”라고 출력한 뒤 continue로 다시 메뉴 선택 화면으로 돌아가도록 처리하였다.

### **🛠️ Troubleshooting & 기술 회고:**
문제 1: 함수로 나눈 뒤 사용자 정보가 여러 기능에서 공유되어야 했다.
원인: 이번에 코딩할 때 사용자 입력, BMI 확인, 목표 체중 확인, 저장 정보 출력 기능을 각각 함수로 분리하였다. 그런데 사용자 정보를 특정 함수 안에서만 만들면 다른 함수에서 그 정보를 사용할 수 없었다.
해결: 프로그램 시작 부분에 users = [] 리스트를 먼저 만들고, 사용자 정보를 입력받는 함수에서 이 리스트에 데이터를 추가하도록 수정하였다. 이를 통해 여러 함수가 같은 사용자 정보 목록을 사용할 수 있게 하였다.
문제 2: 여러 명의 사용자 정보를 저장하기 위한 데이터 구조가 필요했다.
원인: 기존 프로그램은 한 명의 사용자 정보만 저장하는 구조였기 때문에 새로운 사용자를 입력하면 이전 정보가 유지되기 어려웠다. V4.0에서는 여러 명의 사용자 정보를 관리해야 했기 때문에 데이터 저장 방식을 확장해야 했다.
해결: 각 사용자의 정보를 [이름, 몸무게, 키, 목표 체중] 형태의 리스트로 저장하고, 이 리스트를 users 리스트 안에 추가하였다. 이를 통해 여러 사용자의 정보를 2차원 리스트 구조로 관리할 수 있게 하였다.
문제 3: BMI 계산 코드가 여러 기능에서 반복될 수 있었다.
원인: BMI 정보 출력 기능과 저장된 정보 출력 기능 모두 BMI 계산이 필요했다. 같은 계산식을 여러 곳에 직접 작성하면 코드가 길어지고, 나중에 수정할 때 실수할 가능성이 커진다.
해결: BMI 계산식을 calculate_bmi() 함수로 따로 분리하였다. 이후 BMI가 필요한 기능에서는 calculate_bmi(weight, height) 함수를 호출하여 값을 구하도록 수정하였다.
